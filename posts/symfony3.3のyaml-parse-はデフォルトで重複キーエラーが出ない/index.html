Symfony3.3のYaml::parse()はデフォルトで重複キーエラーが出ない

キーが重複したYAMLを、Symfony3.3のYaml::parse()でパースするとサイレントで片方の値が消滅する話

問題

$text = <<<YAML
hoge: 300
foo:  400
bar:  500
foo:  600
YAML;

var_dump(Symfony\Component\Yaml\Yaml::parse($text));

これは foo の重複キーエラーが出るかな、と思いきや出ない。

% php test.php
test.php:12:
array(3) {
  'hoge' =>
  int(300)
  'foo' =>
  int(400)
  'bar' =>
  int(500)
}

どうやら重複した場合は 最初に入力された値 が使われている。

原因

パース処理を追った所、エラー制御演算子によってエラーが表示されていないようだった

https://github.com/symfony/yaml/blob/3.3/Parser.php#L340

- 重複キーを勝手にマッピングするのは3.2からは非推奨ですよ
- 4.0からは `\Symfony\Component\Yaml\Exception\ParseException` が出ますよ

という感じ

対策

とりあえずエラーを出したかったので E_USER_DEPRECATED に set_error_handler() を行うことで対処した

$text = <<<YAML
hoge: 300
foo:  400
bar:  500
foo:  600
YAML;

set_error_handler(
    function ($errno, $errstr, $errfile, $errline) {
        throw new \ErrorException(
            $errstr,
            0,
            $errno,
            $errfile,
            $errline
        );
    },
    E_USER_DEPRECATED
);
var_dump(Symfony\Component\Yaml\Yaml::parse($text));
restore_error_handler();

% php test.php
PHP Fatal error:  Uncaught ErrorException: Duplicate key "foo" detected whilst parsing YAML. Silent handling of duplicate mapping keys in YAML is deprecated since version 3.2 and will throw \Symfony\Component\Yaml\Exce
ption\ParseException in 4.0. in vendor\symfony\yaml\Parser.php:340
